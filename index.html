Practicas JS / Errores Comunes

Click en botones con SVG interno

Cuando un <button> contiene un SVG o icono, el e.target del click sera el SVG o el <path>, NO el boton. Comparar directamente e.target === element fallara.

❌ INCORRECTO
document.addEventListener('click', (e) => { if (e.target !== searchToggle) { // falla si click fue en SVG/path searchExpand.classList.remove('open'); } });
✓ CORRECTO
document.addEventListener('click', (e) => { if (!searchToggle.contains(e.target)) { // verifica si click fue dentro del boton searchExpand.classList.remove('open'); } });

Regla general: Siempre usar element.contains(e.target) en lugar de e.target === element cuando el elemento destino contiene hijos (SVGs, spans, iconos). Aplica a: botones con iconos, dropdowns, popovers, modales con botones internos.
